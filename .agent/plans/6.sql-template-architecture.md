# Plan 6: SQL Template Architecture

⚠️ **Medium** — New module pre-built for Module 7. No wiring into chat until Module 7.

## Problem

Module 7 will add a text-to-SQL tool. Free-form SQL generation by LLMs is dangerous in production:
- Prompt injection can exfiltrate data despite RLS
- Inefficient LLM-generated JOINs on large tables can DoS the database
- No way to enforce read-only queries at the LLM output level

## Solution

The LLM never writes SQL. It picks from a hardcoded allowlist of safe, parameterized, read-only templates and fills in parameters only.

```
Fragile:  user asks → LLM writes SQL → execute
Safe:     user asks → LLM picks template + fills params → execute pre-approved SQL
```

## Template Allowlist (8 templates)

| Template | Purpose | Params |
|---|---|---|
| `count_documents` | How many docs uploaded | none |
| `list_documents` | All docs with status + chunk count | none |
| `count_chunks_by_document` | Chunks per doc | none |
| `find_documents_by_status` | Filter by status | `status` |
| `search_chunks_by_keyword` | ILIKE keyword search across chunks | `keyword` |
| `get_recent_conversations` | Recent conversation list | none |
| `count_messages_in_conversation` | Message count by title fragment | `title_fragment` |
| `storage_summary` | Total bytes used | none |

## Security Properties

- `user_id` is **always injected server-side** — never accepted from the LLM
- Template name validated against `ALLOWED_TEMPLATES` set before execution
- Each Supabase RPC is `SET transaction_read_only = on` + `SET statement_timeout = '5s'`
- LLM tool schema exposes `template_name` as an `enum` — can only name valid templates
- `TemplateCall` Pydantic model validates structure before `execute_query` is called

## Supabase Setup (run before Module 7)

Run `supabase/migrations/004_sql_query_rpcs.sql` in the Supabase SQL editor.

Creates 8 `SECURITY DEFINER` RPC functions, each:
- Read-only (`SET transaction_read_only = on`)
- 5-second timeout (`SET statement_timeout = '5s'`)
- Accepts `p_user_id UUID` as first argument (always user-scoped)
- Granted to `service_role` only
