# Module 3: Record Manager

✅ **Simple** — Single-pass executable. Localized to one migration, one backend router, one model, and one frontend page. SHA-256 is stdlib; no new dependencies.

## Context

Naive ingestion allows the same file to be uploaded repeatedly, creating duplicate chunks that bloat the vector store and inflate embedding costs. This module adds content hashing to detect duplicates and enable incremental updates.

**Goal:**
- Upload same content → reject with 409 and tell user which existing file has that content
- Upload same filename with different content → delete old document + chunks, re-ingest new version
- Upload new file → works as before

## Key Design Decisions

1. **Hash at router layer, not ingestion_service** — dedup check must happen synchronously before any storage write so a 409 can be returned immediately
2. **Partial unique index on `completed` only** — allows retry of failed ingestion, no false conflicts during pending/processing window
3. **Delete-then-insert for updates** — deleting the old document row cascades to chunks atomically; new row processed through existing pipeline unchanged
4. **Hash from raw bytes, not decoded text** — consistent regardless of encoding normalisation; same binary always = same hash
