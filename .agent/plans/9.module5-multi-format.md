# Module 5: Multi-Format Support

⚠️ **Medium** — No DB migration needed. Docling install is large (~2-3GB with PyTorch). Key
architectural risk: docling parsing is blocking/CPU-bound and must run off the async event loop.

## Context

Modules 1–4 built auth, RAG chat, deduplication, and metadata extraction. Ingestion currently
accepts only `.txt` and `.md`. Module 5 adds PDF, DOCX, and HTML by inserting a **parsing layer**
before the existing chunking → embedding → metadata pipeline. The pipeline downstream stays
completely unchanged — it receives a plain text string regardless of source format.

**FK cascade deletes are already in place** from migration 002 (`document_chunks` has
`on delete cascade`). No DB migration required for Module 5.

**Prerequisite:** Migration `009_document_metadata.sql` must be applied in the Supabase SQL editor
before Module 5 end-to-end testing (it adds the `metadata` column the ingestion service writes to).

## Architecture

```
upload endpoint
    │
    ├─ validate MIME / extension
    ├─ read bytes → compute content_hash (raw bytes, pre-parse)
    ├─ check duplicate / incremental update
    ├─ store raw file to Supabase Storage
    ├─ insert documents row (status: pending)
    ├─ asyncio.to_thread(parse_document, bytes, filename, mime)  ← NEW
    │       └─ returns plain text string
    └─ create_task(process_document(id, user_id, text))
           └─ existing pipeline unchanged
```

**Why `asyncio.to_thread`:** docling runs ML inference (CPU-bound, blocking). Calling it directly
in an `async` FastAPI endpoint blocks the event loop and starves all other requests. Moving it to
a thread pool keeps the server responsive.

**Why parse before the background task:** `process_document` is already a background task that
runs after the endpoint returns. If parsing fails (corrupted file, unsupported encoding) we want
that error surfaced to the user immediately as an HTTP 422, not silently swallowed in a background
job that just marks the document `failed`.

## Uncertainties / Flags

| # | Uncertainty | Mitigation |
|---|-------------|------------|
| 1 | First docling call downloads model weights at runtime (~500MB). Could appear as a hang on the first upload. | Document this in the plan; warn user to test with a small file first. |
| 2 | Browsers sometimes send DOCX as `application/octet-stream`. | Extension-based fallback in both router and parsing service. |
| 3 | Docling may produce empty markdown for a scanned PDF (image-only, no OCR layer). | Check `text_content.strip()` before passing to `process_document`; surface 422 if empty. |
| 4 | `DocumentConverter()` re-loads config on every instantiation but reuses cached model weights. If this proves slow under load, a module-level singleton can be introduced later. For now, one instance per parse call is the safe, simple choice. | Note in code comment; revisit in Module 6/7 if latency is a problem. |

### Task 2 — Create `parsing_service.py`

Create `backend/app/services/parsing_service.py`. This module has **one job**: bytes → text. It
knows nothing about documents, users, FastAPI, or Supabase. Keep it pure.

```python
import os
import tempfile
from pathlib import Path

_TEXT_MIME = {"text/plain", "text/markdown"}
_TEXT_EXT  = {".txt", ".md"}

_DOCLING_MIME = {
    "application/pdf",
    "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
    "text/html",
}
_DOCLING_EXT = {".pdf", ".docx", ".html", ".htm"}

_MIME_TO_EXT = {
    "application/pdf":  ".pdf",
    "application/vnd.openxmlformats-officedocument.wordprocessingml.document": ".docx",
    "text/html":        ".html",
}


def parse_document(content: bytes, filename: str, mime_type: str) -> str:
    """Convert raw file bytes to plain text.

    Plain text and markdown are decoded directly.
    PDF, DOCX, and HTML are parsed via docling (blocking — call via asyncio.to_thread).

    Raises:
        ValueError: unsupported file type
        RuntimeError: docling conversion failed or produced empty output
    """
    ext = Path(filename).suffix.lower()

    if mime_type in _TEXT_MIME or ext in _TEXT_EXT:
        return content.decode("utf-8")

    if mime_type in _DOCLING_MIME or ext in _DOCLING_EXT:
        resolved_ext = ext if ext in _DOCLING_EXT else _MIME_TO_EXT.get(mime_type, ".bin")
        text = _parse_with_docling(content, resolved_ext)
        if not text.strip():
            raise RuntimeError(
                "Document parsed but produced no text. "
                "If this is a scanned PDF, it may be image-only and not supported."
            )
        return text

    raise ValueError(f"Unsupported file type: mime={mime_type!r}, filename={filename!r}")


def _parse_with_docling(content: bytes, ext: str) -> str:
    # Import here so startup isn't slowed by docling if it's never called
    from docling.document_converter import DocumentConverter

    with tempfile.NamedTemporaryFile(suffix=ext, delete=False) as f:
        f.write(content)
        tmp_path = f.name

    try:
        converter = DocumentConverter()
        result = converter.convert(tmp_path)
        return result.document.export_to_markdown()
    finally:
        os.unlink(tmp_path)
```
