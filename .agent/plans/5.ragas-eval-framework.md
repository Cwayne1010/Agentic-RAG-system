# Plan 5: RAGAS Evaluation Framework

⚠️ **Medium** — New directory, 4 new files, new dependencies. No changes to existing app code.

## Problem

LangSmith traces confirm the pipeline runs but not whether it answers correctly. After any chunking strategy, model, or prompt change, there's no automated way to detect regressions in answer quality.

## Solution

`backend/eval/` with two scripts:
1. `generate_golden_set.py` — auto-generates Q&A pairs from uploaded doc chunks using OpenRouter
2. `run_eval.py` — runs the golden set through the live pipeline and scores with RAGAS

RAGAS metrics scored:
- **Faithfulness** — does the answer match the retrieved context?
- **Answer Relevancy** — is the answer relevant to the question?
- **Context Precision** — are the retrieved chunks actually useful?

## New Files Created

```
backend/eval/
├── __init__.py
├── golden_set.json          # Q&A ground truth — populate with generate script
├── ragas_config.py          # Wires OpenRouter as judge LLM (no extra API key needed)
├── generate_golden_set.py   # Run once per document set to build golden_set.json
└── run_eval.py              # Run after any pipeline change to score quality
```

## New Dependencies

```bash
cd backend && source venv/bin/activate

```

## New Environment Variables

Add to `backend/.env`:
```
RAGAS_JUDGE_MODEL=openai/gpt-4o-mini
RAGAS_EMBEDDING_MODEL=text-embedding-3-small
```

## Usage

### Step 1: Upload documents and wait for status: completed

### Step 2: Generate golden set
```bash
cd backend && source venv/bin/activate
python -m eval.generate_golden_set \
    --user-id <your-supabase-user-uuid> \
    --output eval/golden_set.json \
    --pairs-per-chunk 2 \
    --max-chunks 20
```
Generates ~40 Q&A pairs from 20 chunks. Commit `golden_set.json` to the repo.

### Step 3: Run evaluation (after any pipeline change)
```bash
python -m eval.run_eval \
    --user-id <uuid> \
    --golden-set eval/golden_set.json \
    --output eval/results.json
```

### Expected output:
```
Faithfulness:      0.85  (does answer match retrieved context?)
Answer Relevancy:  0.79  (is answer relevant to the question?)
Context Precision: 0.91  (are retrieved chunks actually useful?)
Full results → eval/results.json
```

## How RAGAS Uses OpenRouter

`ragas_config.py` wraps `langchain_openai.ChatOpenAI` (pointed at OpenRouter's base URL) with RAGAS's `LangchainLLMWrapper`. The judge model scores each answer against the ground truth. RAGAS's internal use of LangChain is eval-only — it does not touch the app's data pipeline.

## Validation

```bash
# Pass criteria:
# 1. generate_golden_set.py runs without error, produces golden_set.json with Q&A pairs
# 2. run_eval.py runs without error, produces results.json with 3 scores between 0-1
# 3. Scores make sense: faithfulness > 0.5 on a well-ingested document set
```
