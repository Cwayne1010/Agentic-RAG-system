# Module 4: Metadata Extraction

⚠️ **Medium** — Multi-layer change (DB → backend services → retrieval → frontend), feasible in one pass.

## Context

Modules 1–3 built auth, RAG chat, and deduplication. Retrieval currently returns raw chunk content with no source attribution — the LLM has no context about *which document* a chunk came from, what type it is, or what topics it covers.

Module 4 adds LLM-extracted structured metadata to documents at **ingestion time** (one-time cost per document). At query time, metadata is injected alongside chunks in the context block — giving the LLM richer source attribution with zero extra LLM calls. This preserves the Plan 4 architectural decision: 1 LLM call per chat turn.

The RPC is also updated to support optional `filter_doc_type` filtering, enabling future modules to scope retrieval to specific document types.

## Metadata Schema

Pydantic `DocumentMetadata` model extracted per document:

| Field | Type | Example |
|-------|------|---------|
| `doc_type` | `str` | `"contract"`, `"report"`, `"email"`, `"technical"`, `"article"`, `"other"` |
| `language` | `str` | `"en"` |
| `topics` | `list[str]` | `["AI", "employment", "gdpr"]` |
| `summary` | `str` | One or two sentence document summary |
| `entities` | `list[str]` | Named people, orgs, places |
| `date` | `str \| None` | ISO date if found in document, else `null` |

## Task List

### Task 1 — DB Migration (`009_document_metadata.sql`)

Add `metadata jsonb` column to `documents` table. Drop and recreate `match_document_chunks` RPC to JOIN `documents`, return `doc_filename` and `doc_metadata`, and accept optional `filter_doc_type text DEFAULT NULL`.

New RPC signature:

```sql
create or replace function match_document_chunks(
  query_embedding vector(1536),
  match_user_id uuid,
  match_count int default 5,
  filter_doc_type text default null
)
returns table (
  id uuid, document_id uuid, content text, chunk_index int,
  similarity float, doc_filename text, doc_metadata jsonb
)
```

Filter clause: `AND (filter_doc_type IS NULL OR d.metadata->>'doc_type' = filter_doc_type)`

**Files:**

- Create: `supabase/migrations/009_document_metadata.sql`

**Validation:** Paste into Supabase SQL editor, confirm column and RPC created without errors.

### Task 2 — Metadata Extraction Service

Create `backend/app/services/metadata_service.py`:

- `DocumentMetadata` Pydantic model (6 fields above)
- `async def extract_metadata(text: str) -> dict` — uses first 2000 chars

- Uses `get_settings()` for model/API key (same pattern as `openrouter_service.py`)

### Task 4 — Update Retrieval Service

Add `doc_type_filter: str | None = None` parameter to `retrieval_service.search()`. Pass it to the RPC only when non-null:

```python
async def search(query: str, user_id: str, top_k: int = 5, doc_type_filter: str | None = None) -> list[dict]:
    ...
    params = {"query_embedding": ..., "match_user_id": user_id, "match_count": top_k}
    if doc_type_filter:
        params["filter_doc_type"] = doc_type_filter
    result = supabase.rpc("match_document_chunks", params).execute()
```

### Task 5 — Enrich Context Block in Chat Router

`chat.py` currently formats chunks as `[Chunk N | similarity X]: content`. Update to include source metadata from the new RPC return fields:

```
[Source: {doc_filename} | Type: {doc_type} | Chunk {chunk_index} | similarity {similarity:.2f}]: {content}
```

No extra LLM calls — this runs synchronously within the existing pre-fetch step established in Plan 4.

Also update the retrieval SSE event to include `doc_filename` of unique sources found:
